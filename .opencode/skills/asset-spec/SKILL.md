# 资产规格生成技能

从设计文档生成详细的资产规格文件。

---

## ⚠️ 执行规则

1. **先读后问** — 在提问前先读取所有源文档
2. **确认资产列表** — 不经用户确认不开始规格生成
3. **锚定美术圣经** — 每个规格必须引用美术圣经规则
4. **立即更新清单** — 规格批准后立即更新资产清单

---

## 用法

```
/asset-spec system:tower-defense
/asset-spec level:iron-gate-fortress
/asset-spec character:frost-warden
```

无参数时：检查 `design/assets/asset-manifest.md`，找到下一个未规格化的上下文。

---

## Phase 1: 收集上下文

### 必读文档

**美术圣经**: 读取 `design/art/art-bible.md` — 如果不存在：
> "未找到美术圣经。请先运行 `/art-bible` — 资产规格必须锚定到美术圣经的视觉规则。"

提取：视觉身份声明、色彩系统、形状语言、资产标准（第 8 部分）。

**技术偏好**: 读取 `.opencode/docs/technical-preferences.md` — 提取性能预算和命名规范。

### 按目标类型读取源文档

| 目标类型 | 读取路径 | 提取内容 |
|----------|----------|----------|
| **system** | `design/gdd/[target].md` | 视觉/音频需求部分 |
| **level** | `design/levels/[target].md` | 美术需求、资产列表、VFX 需求 |
| **character** | `design/narrative/characters/[target].md` | 视觉描述、角色、区分特征 |

### 呈现上下文摘要

```
资产规格: [目标类型] — [目标名称]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
源文档: [路径] — [N] 种资产类型
美术圣经: 已找到 — 资产标准在第 8 部分
已有规格: [N 个已规格 / 无]
共享资产: [列表或"无"]
```

---

## Phase 2: 资产识别

从源文档提取每种资产类型 — 显式和隐式的。

**系统**: VFX 事件、精灵引用、UI 元素、音频触发、粒子效果、图标需求
**关卡**: 独特环境道具、氛围 VFX、灯光设置、环境音频、天空盒
**角色**: 精灵表（闲置、行走、攻击、死亡）、头像、能力 VFX、UI 表示

### 资产分类

| 类别 | 内容 |
|------|------|
| **Sprite / 2D 美术** | 角色精灵、UI 图标、瓦片表 |
| **VFX / 粒子** | 击中效果、氛围粒子、屏幕效果 |
| **Environment** | 道具、瓦片、背景、天空盒 |
| **UI** | HUD 元素、菜单美术、字体 |
| **Audio** | 音效、音乐、环境循环 *(仅描述，无生成提示词)* |
| **3D Assets** | 网格、材质（如适用） |

### 确认资产列表

**必须**先确认资产列表：

```
我为 [目标] 识别了 [N] 个资产，分布在 [N] 个类别：

📋 Sprite/2D: [列表]
✨ VFX: [列表]
🌿 Environment: [列表]
🎮 UI: [列表]
🔊 Audio: [列表]

A) 继续 — 规格化这些资产
B) 移除一些资产
C) 添加我漏掉的资产
D) 调整分类
```

**不经确认不进入 Phase 3。**

## Phase 3: 规格生成

### 规格模板

```markdown
## ASSET-[NNN] — [资产名称]

| 字段 | 值 |
|------|-----|
| Category | [Sprite / VFX / Environment / UI / Audio / 3D] |
| Dimensions | [例: 256×256px, 4帧精灵表] |
| Format | [PNG / SVG / WAV 等] |
| Naming | [例: vfx_frost_hit_01.png] |
| Polycount | [如果是 3D — 例: <800 tris] |
| Texture Res | [例: 512px — 匹配美术圣经 §8 Tier 2] |

**视觉描述:**
[2-3 句话。足够具体让两个美术师产出一致结果。]

**美术圣经锚点:**
- §3 形状语言: [应用的相关规则]
- §4 色彩系统: [色彩角色 — 例 "使用威胁蓝，按语义色规则"]

**生成提示词:**
[可直接用于 AI 图像工具的提示词。包含：风格关键词、构图说明、色板锚点、灯光方向、负面提示词。]

**Status:** Needed
```

### 生成内容

对于每个资产：

1. **视觉描述** (2-3 句)
   - 锚定美术圣经的形状语言和色彩系统
   - 足够具体让两个美术师产出一致结果

2. **美术圣经锚点**
   - 引用具体章节和规则

3. **AI 生成提示词**
   - 风格关键词
   - 构图说明
   - 色板锚点
   - 灯光方向
   - 负面提示词

## Phase 4: 审核与批准

呈现所有规格后：

```
资产规格 [目标] — [N] 个资产。审核完成？

A) 批准全部 — 写入文件
B) 修改特定资产
C) 用不同方向重新生成
```

---

## Phase 5: 写入规格文件

批准后写入：`design/assets/specs/[target-name]-assets.md`

```markdown
# 资产规格 — [目标类型]: [目标名称]

> **源文档**: [GDD/关卡/角色文档路径]
> **美术圣经**: design/art/art-bible.md
> **生成日期**: [日期]
> **状态**: [N] 个资产已规格 / [N] 个已批准 / [N] 个制作中 / [N] 个完成

---

[所有资产规格...]
```

### 更新资产清单

创建或更新 `design/assets/asset-manifest.md`：

```markdown
# 资产清单

> 最后更新: [日期]

## 进度摘要

| 总计 | 需求中 | 制作中 | 完成 | 已批准 |
|------|--------|--------|------|--------|
| [N] | [N] | [N] | [N] | [N] |

## 按上下文分类的资产

### [目标类型]: [目标名称]
| 资产 ID | 名称 | 类别 | 状态 | 规格文件 |
|---------|------|------|------|----------|
| ASSET-001 | [名称] | [类别] | Needed | specs/[target]-assets.md |
```

## 资产 ID 分配

资产 ID 在整个项目中顺序分配 — 不是按上下文。

读取清单找到当前最高编号：
```
grep "ASSET-" design/assets/asset-manifest.md
```

新资产从 `ASSET-[最高+1]` 开始。

如果清单不存在，从 `ASSET-001` 开始。

---

## 共享资产协议

规格化前检查是否有等效资产已在其他上下文规格化：

- 通用 UI 元素（血条、分数显示）
- 通用环境道具
- 角色 VFX（击中火花、死亡效果）

如果找到匹配：引用现有 ASSET-ID 而非创建重复。

> "ASSET-012（通用击中火花）已为战斗系统规格化。为塔防重用 — 添加 tower-defense 到引用列表。"

```
资产规格 [目标] 完成。下一步？

A) 规格化另一个系统 — /asset-spec system:[next]

C) 规格化一个角色 — /asset-spec character:[name]
D) 运行 /asset-audit — 验证交付资产与规格
E) 完成
```

---

## 协作协议

每个阶段遵循：**识别 → 确认 → 生成 → 审核 → 批准 → 写入**

- 不经确认不规格化
- 始终锚定美术圣经
- 明确指出所有冲突
- 只在批准后写入规格文件
- 写入后立即更新清单
