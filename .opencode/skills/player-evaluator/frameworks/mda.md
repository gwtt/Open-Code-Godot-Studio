# MDA 框架 (Mechanics-Dynamics-Aesthetics)

游戏设计的三个抽象层次。

---

## 框架概述

**MDA** 将游戏分解为三个层次:

```
玩家视角:    Aesthetics → Dynamics → Mechanics
设计者视角:  Mechanics → Dynamics → Aesthetics
```

**设计者从 Mechanics 出发，玩家从 Aesthetics 体验。**

---

## 第一层: Mechanics (机制)

**定义**: 游戏的规则、数据和算法。

**内容**:
- 游戏规则
- 操作方式
- 数值系统
- 算法逻辑

**示例**:
- 跳跃高度 = 力量 × 重力
- 伤害 = 攻击力 - 防御力
- 移动速度 = 基础速度 + 加成

**设计检查**:
- [ ] 规则是否清晰?
- [ ] 数值是否平衡?
- [ ] 机制是否服务于体验?

---

## 第二层: Dynamics (动态)

**定义**: 机制在玩家交互中产生的运行时行为。

**内容**:
- 玩家策略
- 涌现玩法
- 系统交互
- 社会动态

**示例**:
- 机制: 资源有限 → 动态: 竞争/合作策略
- 机制: 技能冷却 → 动态: 攻防节奏
- 机制: 交易系统 → 动态: 经济行为

**设计检查**:
- [ ] 机制是否产生预期动态?
- [ ] 是否有负面涌现?
- [ ] 策略是否多样?

---

## 第三层: Aesthetics (美学体验)

**定义**: 玩家与游戏交互时的情感反应。

**8 种美学类型** (Hunicke et al.):

| 美学 | 描述 | 示例游戏 |
|------|------|----------|
| **感知** | 感官愉悦 | 音游、视觉小说 |
| **叙事** | 故事体验 | RPG、冒险游戏 |
| **挑战** | 克服困难 | 魂系列、解谜 |
| **探索** | 发现未知 | 开放世界 |
| **社交** | 社交互动 | MMO、派对游戏 |
| **表达** | 自我表达 | 沙盒、建造游戏 |
| **休闲** | 放松消遣 | 休闲游戏 |
| **幻想** | 扮演体验 | RPG、模拟游戏 |

**设计检查**:
- [ ] 目标美学是什么?
- [ ] 机制是否支持美学?
- [ ] 玩家实际体验是否匹配?

---

## 设计视角转换

### 设计者 → 玩家

```
设计者定义:
  Mechanics (跳跃机制)
    → Dynamics (平台跳跃挑战)
      → Aesthetics (成就感、紧张感)

玩家体验:
  Aesthetics (我感到紧张和成就感)
    → Dynamics (跳跃时机很重要)
      → Mechanics (按下跳跃键)
```

### 评估视角

**玩家视角评估** (本 skill 使用):
1. **Aesthetics**: 我感受到什么?
2. **Dynamics**: 游戏行为是什么样的?
3. **Mechanics**: 这背后的规则是什么?

---

## 反向设计法

从 Aesthetics 出发:

```
1. 定义目标体验 (Aesthetics)
   "玩家应该感到紧张刺激"

2. 识别产生该体验的动态 (Dynamics)
   "紧张来自时间压力和惩罚"

3. 设计支持动态的机制 (Mechanics)
   "时间限制 + 死亡惩罚"
```

**示例**:
| 目标美学 | 所需动态 | 支持机制 |
|----------|----------|----------|
| 紧张刺激 | 时间压力、风险 | 倒计时、惩罚 |
| 放松享受 | 无压力、自由 | 无时间限制、沙盒 |
| 成就感 | 挑战、进步 | 难度、成长系统 |

---

## 评估问题

从玩家视角评估游戏概念时:

### Aesthetics 层
1. 目标情感体验是什么?
2. 玩家最可能感受到什么?
3. 是否有情感冲突?

### Dynamics 层
1. 玩家行为会是什么样的?
2. 是否会产生意外动态?
3. 策略空间是否足够?

### Mechanics 层
1. 规则是否支持预期动态?
2. 数值是否平衡?
3. 是否有冗余机制?

---

## MDA 与其他框架整合

| 框架 | 聚焦点 | 与 MDA 关系 |
|------|--------|-------------|
| **Bartle** | 玩家类型 | 不同玩家对不同 Aesthetics 有偏好 |
| **Flow** | 心流状态 | Dynamics 需要支持心流条件 |
| **SDT** | 动机需求 | Aesthetics 满足三种心理需求 |
